# Баг-репорты

## BUG-01: Фильтр диапазона цен (TC-D-01)

| Поле | Значение |
|------|-----------|
| **Приоритет** | High |
| **Автотест** | `test_price_filter` |
| **Предусловия** | Открыта главная страница, список объявлений загружен |
| **Шаги** | 1. Ввести минимальную цену 1000<br>2. Ввести максимальную цену 30000<br>3. Дождаться обновления информации в выдаче |
| **Ожидаемый результат** | В выдаче отображаются только объявления с ценой в указанном диапазоне |
| **Фактический результат** | В выдаче присутствуют объявления с ценой 32456, 40000, 500 (вне диапазона) |

## BUG-02: Тогл «Только срочные» (TC-D-05)

| Поле | Значение |
|------|-----------|
| **Приоритет** | Medium |
| **Автотест** | `test_urgency_toggle_on` |
| **Предусловия** | Открыта главная страница, список объявлений загружен |
| **Шаги** | 1. В выпадающем списке «Приоритет» выбрать «Срочный»<br>2. Дождаться обновления информации в выдаче<br>3. Проверить выдачу |
| **Ожидаемый результат** | При включении - только срочные объявления |
| **Фактический результат** | В выдаче присутствуют объявления без метки «Срочно» (№3,7,8,10) |

## BUG-03: Страница статистики: запуск таймера (TC-D-08)

| Поле | Значение |
|------|-----------|
| **Приоритет** | Medium |
| **Автотест** | `test_stats_start_timer` |
| **Предусловия** | Открыта страница статистики, таймер остановлен |
| **Шаги** | 1. Нажать кнопку запуска таймера<br>2. Проверить, что отсчет таймера начался с 5 минут |
| **Ожидаемый результат** | Автоматическое обновление возобновляется, таймер запущен |
| **Фактический результат** | Таймер не запускается, надпись «Автообновление выключено» не исчезает |
